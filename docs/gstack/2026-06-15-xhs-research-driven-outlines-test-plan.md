Generated by gstack `/plan-eng-review` on 2026-06-15

## Affected Pages And Routes

- `POST /xhs-analysis/research/outlines` — validates the full research to outline
  pipeline.
- `GET /conversations/:id` — restores latest visible research summary.
- `packages/web-frontend/app/page.tsx` — main workbench outline generation flow.
- `packages/web-frontend/app/workbench/outline-workspace.tsx` or a new research
  summary component — compact research evidence above outlines.

## Key Interactions To Verify

- User enters an idea and generates outlines.
- User clicks `换一批`; old outline batches remain visible.
- User sees low-confidence warning when samples are sparse.
- User sees compliant source metadata without raw copied body text.
- User opens a saved conversation and sees latest research summary restored.

## Edge Cases

- Empty idea.
- Missing custom provider config.
- One keyword search fails.
- All connector calls fail due network/setup error.
- Connector returns valid empty result.
- Connector returns malformed wrapper.
- Duplicate notes across keywords.
- Double-click generate while request is in flight.

## Critical Paths

1. Authenticated user owns conversation -> research endpoint creates run and
   outline batch.
2. Partial connector failure -> completed with warnings and editable outlines.
3. Sparse or empty valid provider result -> fallback/low-confidence editable
   outlines.
4. Provider setup failure -> clear setup error, no misleading research summary.
5. Conversation restore -> latest summary and outline batches render together.
